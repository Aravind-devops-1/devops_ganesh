# Minikube Setup on Windows 


## 1. Introduction

Minikube is a local Kubernetes (K8s) cluster that helps beginners and DevOps engineers learn, test, and practice Kubernetes concepts safely on their laptops.

In this setup:

* OS: **Windows**
* Container Runtime: **Docker Desktop**
* Linux Backend: **WSL2**
* Kubernetes Cluster: **Minikube (single-node)**

This guide assumes:

* Docker Desktop is **already installed and running**
* WSL2 is **already enabled**
* You want a **stable DevOps-style setup**


## 2. What Terminal to Use (IMPORTANT)

| Task                         | Terminal   | Admin Required |
| ---------------------------- | ---------- | -------------- |
| Download Minikube            | PowerShell | No           |
| Verify Docker                | PowerShell | No           |
| Start Minikube               | PowerShell | No           |
| Fix permission / path issues | PowerShell | Sometimes    |

**Recommendation:**
 * Use **PowerShell (NOT Command Prompt)**
 * Run **without Administrator** unless mentioned explicitly



## 3. Pre-checks (Must Do Before Installing)

### 3.1 Verify Docker Desktop is Running

Open **PowerShell (Normal)** and run:

```powershell
docker version
```

Expected Output:

* Client and Server details shown
* No errors

Error:

```
error during connect: docker daemon not running
```

Fix:

* Open **Docker Desktop**
* Wait until status shows **"Docker Desktop is running"**


### 3.2 Verify Docker Context

```powershell
docker context ls
```

Expected:

```
* default
```

If not default:

```powershell
docker context use default
```

### 3.3 Verify WSL Version

```powershell
wsl --status
```

Expected:

* Default Version: **2**

If not:

```powershell
wsl --set-default-version 2
```


## 4. Download Minikube (Windows)

### 4.1 Download Minikube Binary

Run in **PowerShell (Normal)**:

```powershell
curl -LO https://storage.googleapis.com/minikube/releases/latest/minikube-windows-amd64.exe
```

---

### 4.2 Move Minikube to a Directory in PATH

Run **PowerShell as Administrator** ONLY for this step:

```powershell
mkdir C:\minikube
move minikube-windows-amd64.exe C:\minikube\minikube.exe
```

Add Minikube to PATH:

```powershell
setx PATH "$env:PATH;C:\minikube" /M
```

Close **ALL PowerShell windows** and reopen a new one.

---

### 4.3 Verify Minikube Installation

Open **PowerShell (Normal)**:

```powershell
minikube version
```

Expected Output:

```
minikube version: vX.Y.Z
```

Error:

```
minikube is not recognized
```

Fix:

* PATH not updated
* Restart PowerShell
* Verify `C:\minikube` is in Environment Variables

## 5. Start Minikube Using Docker Driver (Recommended)

### 5.1 Start Cluster

```powershell
minikube start --driver=docker
```

What this does:

* Uses Docker Desktop as runtime
* Avoids VirtualBox/Hyper-V conflicts
* Best for DevOps learners

### 5.2 Expected Success Output

Key lines you should see:

* `Using Docker driver`
* `Kubernetes control plane running`
* `kubectl configured`

## 6. Install kubectl (If Not Installed)

Minikube can install kubectl automatically:

```powershell
minikube kubectl -- get nodes
```

If you want standalone kubectl:

```powershell
minikube kubectl -- version
```



## 7. Verify Kubernetes Cluster

### 7.1 Check Cluster Status

```powershell
minikube status
```

Expected:

```
host: Running
kubelet: Running
apiserver: Running
```


### 7.2 Check Nodes

```powershell
kubectl get nodes
```

Expected:

```
minikube   Ready   control-plane
```

## 8. Common Errors and Fixes

### Error: Docker Driver Not Found

```
Exiting due to DRV_NOT_FOUND
```

Fix:

* Open Docker Desktop
* Enable:

  * Settings → General → **Use WSL 2 based engine**
* Restart Docker Desktop


### Error: Kubernetes Not Ready

```
apiserver not responding
```

Fix:

```powershell
minikube stop
minikube delete
minikube start --driver=docker
```

### Error: Port Already in Use

```
bind: address already in use
```

Fix:

```powershell
netstat -ano | findstr :8443
taskkill /PID <PID> /F
```

Then restart Minikube.
