### Add a Taint to the Minikube Node

Minikube runs as a **single node** named `minikube`.

Add a taint:

```bash
kubectl taint nodes minikube env=prod:NoSchedule
```

Meaning:

* `env=prod` → taint key/value
* `NoSchedule` → Pods **will not be scheduled** unless they tolerate it

---

### Verify the Taint

```bash
kubectl describe node minikube | grep -i taint
```

Expected output:

```
Taints: env=prod:NoSchedule
```

---

### Pod WITHOUT Toleration (Will Fail)

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: no-toleration-pod
spec:
  containers:
  - name: nginx
    image: nginx
```

Apply it:

```bash
kubectl apply -f no-toleration.yaml
```

Check status:

```bash
kubectl get pod no-toleration-pod
```

Result:

```
Pending
```

Reason:

> Pod does NOT tolerate the taint → Scheduler blocks it

---

### Pod WITH Toleration (Will Work)

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: toleration-pod
spec:
  tolerations:
  - key: "env"
    operator: "Equal"
    value: "prod"
    effect: "NoSchedule"
  containers:
  - name: nginx
    image: nginx
```

Apply it:

```bash
kubectl apply -f toleration-pod.yaml
```

Check status:

```bash
kubectl get pod toleration-pod
```

Result:

```
Running
```

---

### What Actually Happened (Beginner View)

* Node has **“No entry” sign** (taint)
* First Pod →  no permission → blocked
* Second Pod →  has permission (toleration) → allowed

But remember:

> **Toleration does NOT force scheduling**
> It only **allows** the Pod to be scheduled **if the scheduler chooses that node**

---

### Clean Up (Important in Minikube)

Remove the taint to avoid blocking future Pods:

```bash
kubectl taint nodes minikube env=prod:NoSchedule-
```
