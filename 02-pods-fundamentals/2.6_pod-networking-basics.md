# Pod Networking Basics

## 1. What Is Pod Networking?

Pod networking defines **how Pods communicate**:

* Inside the same Pod
* With other Pods
* With the outside world (basic view)

> **Key idea:**
> Every Pod gets its **own IP address**.



## 2. Pod IP Address (Core Rule)

* Each Pod is assigned **one unique IP**
* All containers inside the Pod:

  * Share the **same IP**
  * Share the **same network namespace**

```
Pod IP = 10.244.0.5
```


## 3. Container-to-Container Communication (Same Pod)

Containers inside a Pod communicate using:

```
localhost
```

### Example

* App container runs on port `8080`
* Sidecar container can access it using:

```bash
curl http://localhost:8080
```

* No Service required
* No DNS required


## 4. Pod-to-Pod Communication (Cluster Level)

* Pods can communicate with **any other Pod**
* No NAT
* No port mapping required

```
Pod A (10.244.0.5) → Pod B (10.244.1.7)
```

Communication happens via the **cluster network (CNI)**.



## 5. Important Networking Rule (Kubernetes Promise)

> **Every Pod can reach every other Pod directly using its IP**

This holds:

* Across nodes
* Across namespaces (unless restricted)



## 6. Why Pod IPs Are NOT Reliable

Pod IPs are:

* Temporary
* Recreated on restart
* Changed on rescheduling

* Never use Pod IPs directly in applications
* Use **Services** for stable access (covered later)


## 7. How Networking Is Implemented (High Level)

* Kubernetes uses **CNI plugins**
* Examples:

  * Flannel
  * Calico
  * Cilium

You don’t manage this manually in Minikube.



## 8. Hands-On: Observe Pod Networking

### Create a Simple Pod

```bash
kubectl run nginx-pod --image=nginx
```

### Get Pod IP

```bash
kubectl get pod nginx-pod -o wide
```

Output includes:

```
IP: 10.x.x.x
```



### Exec Into the Pod

```bash
kubectl exec -it nginx-pod -- bash
```

Inside the Pod:

```bash
ip addr
hostname -i
```



## 9. Test Pod-to-Pod Communication

Create another Pod:

```bash
kubectl run busybox --image=busybox -- sleep 3600
```

Exec into busybox:

```bash
kubectl exec -it busybox -- sh
```

Ping nginx Pod:

```bash
ping <nginx-pod-ip>
```

This confirms Pod-to-Pod networking



## 10. Common Beginner Mistakes

* Expecting Pod IP to be permanent
* Using Pod IP in config files
* Assuming containers need port mapping inside a Pod

## 11. Key Takeaways

* Each Pod has one IP
* Containers in a Pod use `localhost`
* Pods can talk to each other directly
* Pod IPs are ephemeral
* Services provide stability (next topic)
