# Pod vs Container (Kubernetes Basics)

## What is a Container?

A **container** is a lightweight runtime unit that:

* Packages an application and its dependencies
* Runs a **single process**
* Is created and managed by a container runtime (Docker, containerd)

Example:

* One Nginx container
* One Java application container

Container alone **does not understand Kubernetes**.

## What is a Pod?

A **Pod** is the **smallest deployable unit in Kubernetes**.

A Pod:

* Wraps **one or more containers**
* Provides **shared networking** (one IP)
* Provides **shared storage**
* Has a **single lifecycle**

> Kubernetes never runs containers directly — it runs **Pods**.

---

## Key Difference (At a Glance)

| Aspect           | Container             | Pod                     |
| ---------------- | --------------------- | ----------------------- |
| Level            | Runtime unit          | Kubernetes unit         |
| Runs in          | Container runtime     | Kubernetes              |
| Networking       | Own network namespace | Shared Pod IP           |
| Scaling          | Manual                | Managed via controllers |
| Kubernetes aware | No                    | Yes                     |

---

## Relationship Between Pod and Container

```
Kubernetes
   |
   v
  Pod
   |
   v
Container(s)
```

* A Pod **always contains containers**
* A container **cannot exist alone in Kubernetes**

## Simple Example

* Docker world:

  ```bash
  docker run nginx
  ```

* Kubernetes world:

  ```bash
  kubectl run nginx --image=nginx
  ```

Even here, Kubernetes creates:

```
Pod → Container
```


## When to Think About Containers vs Pods

* Think **Container** when:

  * Building images
  * Writing Dockerfiles

* Think **Pod** when:

  * Deploying apps
  * Debugging Kubernetes
  * Managing workloads


## Important Rule (Interview Favorite)

> **You scale Pods, not containers.**
> **You manage Pods, not containers.**


## Minimal Commands to Observe the Difference

```bash
kubectl get pods
kubectl describe pod <pod-name>
kubectl get pods -o wide
```

These commands show:

* Pod IP
* Containers inside the Pod
* Pod lifecycle state


## One-Line Summary

> A **container runs an application**, but a **Pod is how Kubernetes runs containers**.
