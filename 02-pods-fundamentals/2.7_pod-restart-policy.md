# Pod Restart Policy

## What Is Pod Restart Policy?

A **Pod Restart Policy** defines **what Kubernetes should do when a container inside a Pod stops or crashes**.

It answers one simple question:

> **Should Kubernetes restart the container or not?**



## Restart Policies Available

Kubernetes supports **only three restart policies**:

| Restart Policy | Meaning                         |
| -------------- | ------------------------------- |
| `Always`       | Always restart the container    |
| `OnFailure`    | Restart only if container fails |
| `Never`        | Never restart the container     |



## Default Behavior

* Default restart policy is:

```
Always
```

* If you do not specify it, Kubernetes **automatically uses `Always`**



## Where Restart Policy Is Defined

Restart policy is defined **at Pod level**, not container level.

```yaml
spec:
  restartPolicy: Always
```



## Simple Examples

### 1️. Always (Default)

Used for:

* Web servers
* APIs
* Long-running applications

```yaml
restartPolicy: Always
```

If container crashes → Kubernetes restarts it.


### 2️. OnFailure

Used for:

* Batch jobs
* One-time tasks
* Data processing

```yaml
restartPolicy: OnFailure
```

Restart happens **only if exit code is non-zero**.



### 3️. Never

Used for:

* Debug pods
* One-time manual execution

```yaml
restartPolicy: Never
```

Container stops → Pod stops permanently.



## Important Limitation (Very Important)

**RestartPolicy works only with Pods and Jobs**

| Resource   | Restart Policy |
| ---------- | -------------- |
| Pod        | Yes            |
| Job        | Yes            |
| Deployment | Not directly |
| ReplicaSet | Not directly |

Deployments **always restart containers**, regardless of restartPolicy.


## How to Check Restart Policy

```bash
kubectl get pod <pod-name> -o yaml | findstr restartPolicy
```

or

```bash
kubectl describe pod <pod-name>
```


## How Restart Policy Works Internally (Simple)

```
Container stops
        ↓
Restart Policy checked
        ↓
Restart OR No restart
```


## Common Beginner Mistakes

 * Expecting `Never` to work with Deployments
 * Setting restartPolicy at container level
 * Using `Always` for batch jobs


## Interview One-Liners

* Default restart policy? → **Always**
* Restart policy level? → **Pod**
* Can Deployments stop restarting pods? → **No**


## Key Takeaways

* Restart policy controls container restarts
* Defined at Pod level
* Only three options
* Deployments override restart behavior
