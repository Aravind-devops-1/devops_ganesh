# Pod Storage and Volumes – Basics

## 1. Why Pods Need Storage

By default, containers inside a Pod are **ephemeral**:

* If a container restarts → data is lost
* If a Pod is deleted → data is lost

To solve this, Kubernetes provides **Volumes**.

> **Volumes allow data to survive container restarts and be shared inside a Pod.**

---

## 2. What Is a Volume (Pod Context)

A **Volume** is a directory that:

* Is attached to a Pod
* Can be mounted inside one or more containers
* Exists for the **lifetime of the Pod**

Important:

* Volumes are **defined at Pod level**
* Containers only **mount** them

---

## 3. Pod-Level Storage Behavior

| Scenario           | Data Status                   |
| ------------------ | ----------------------------- |
| Container crashes  | Data remains                  |
| Container restarts | Data remains                  |
| Pod deleted        | Data lost (for basic volumes) |

Advanced persistence is handled later using **PV/PVC**.

---

## 4. Common Volume Types (Beginner)

### 4.1 `emptyDir` (Most Important for Learning)

* Created when Pod starts
* Deleted when Pod is removed
* Shared across containers in the same Pod

Use cases:

* Temporary data
* Cache
* Sharing files between containers

---

### 4.2 `hostPath` (Intro Only)

* Uses a directory from the node filesystem
* Tightly coupled to the node

Not recommended for production
(Used mainly for learning or node-level access)

---

## 5. Simple Pod with Volume Example

### Pod with `emptyDir` Volume

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: volume-demo-pod
spec:
  volumes:
  - name: shared-data
    emptyDir: {}
  containers:
  - name: app
    image: busybox
    command: ["sh", "-c", "echo Hello > /data/message.txt && sleep 3600"]
    volumeMounts:
    - name: shared-data
      mountPath: /data
```

---

## 6. How Volume Mounting Works

```
Pod
 └── Volume (shared-data)
      ├── Container A → /data
      └── Container B → /data
```

* Same data
* Same directory
* Multiple containers can read/write

---

## 7. Verify Volume Behavior (Hands-On)

### Create Pod

```bash
kubectl apply -f volume-pod.yaml
```

### Exec into Pod

```bash
kubectl exec -it volume-demo-pod -- sh
```

### Check Data

```bash
cat /data/message.txt
```

---

## 8. What Volumes Do NOT Solve

* Long-term persistence 
* Cross-Pod data sharing 
* Stateful application storage 

These are solved later using:
 * PersistentVolumes (PV)
 * PersistentVolumeClaims (PVC)

---

## 9. Common Beginner Mistakes

* Expecting data to survive Pod deletion
* Mounting volumes at wrong paths
* Confusing volume lifecycle with container lifecycle

---

## 10. Key Takeaways

* Containers are ephemeral
* Volumes provide Pod-level storage
* Volumes outlive containers, not Pods
* `emptyDir` is best for learning
* Persistence comes later
