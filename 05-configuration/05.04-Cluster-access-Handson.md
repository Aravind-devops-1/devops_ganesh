# Organizing Cluster Access Using kubeconfig Files (Minikube)

## 1. What is a kubeconfig file?

* **kubeconfig** is a **YAML file that stores Kubernetes cluster access information**.
* It tells `kubectl`:

  * Which cluster to connect to
  * Which user credentials to use
  * Which namespace to operate in

**Analogy:**

* Think of kubeconfig as a **remote control**:

  * One remote can control multiple TVs
  * You select which TV (cluster) to operate

---

## 2. Why kubeconfig is important

* Allows **access to multiple clusters** from a single machine.
* Supports **different users and roles**.
* Defines **default namespace** for operations.
* Works with **Minikube** as well as **cloud-managed clusters**.

---

## 3. Minikube Default kubeconfig

When you start Minikube:

```bash
minikube start
```

* Minikube automatically creates a kubeconfig file in:

```
~/.kube/config
```

* This file contains:

  * Cluster info
  * User credentials (client certs)
  * Contexts (cluster + user + namespace combination)

Check current config:

```bash
kubectl config view
```

---

## 4. Key Concepts in kubeconfig

| Term          | Explanation                                                                                    |
| ------------- | ---------------------------------------------------------------------------------------------- |
| **Cluster**   | The Kubernetes API server endpoint + CA certificate                                            |
| **User**      | Credentials (token, client cert) for accessing the cluster                                     |
| **Context**   | Combines a cluster + user + namespace. Switch contexts to change your active cluster/namespace |
| **Namespace** | Logical partition of cluster resources; can be default or custom                               |

---

## 5. Hands-On: Working with kubeconfig in Minikube

### 5.1 View Current Context

```bash
kubectl config current-context
```

* Shows which cluster and namespace you’re currently using.

---

### 5.2 List All Contexts

```bash
kubectl config get-contexts
```

Example output:

```
CURRENT   NAME       CLUSTER    AUTHINFO      NAMESPACE
*         minikube   minikube   minikube      default
```

* `*` indicates the current context.

---

### 5.3 Switch Context

```bash
kubectl config use-context minikube
```

* Useful if you have multiple clusters (Minikube + Cloud cluster).
* You can quickly switch which cluster `kubectl` talks to.

---

### 5.4 Set Namespace for Current Context

```bash
kubectl config set-context --current --namespace=dev
```

* All `kubectl` commands now default to `dev` namespace.

---

### 5.5 Add a New Context (Optional)

```bash
kubectl config set-context test-cluster --cluster=minikube --user=minikube --namespace=test
```

* Defines a new context that points to Minikube cluster under namespace `test`.

---

### 5.6 Access Minikube Cluster with Explicit kubeconfig

```bash
kubectl --kubeconfig=/home/user/.kube/config get pods
```

* Useful if you have multiple kubeconfig files (e.g., for Dev, Prod clusters).

---

## 6. Real-Life Analogy

```
kubeconfig file
 ├─ Cluster = TV
 ├─ User    = Remote control user profile
 ├─ Context = TV + user + default channel (namespace)
 └─ Namespace = Channel/section of TV you want to watch
```

* Switching context = changing TV
* Changing namespace = changing channel

---

## 7. Minikube Tips for Beginners

* Always start Minikube before using `kubectl`.
* Use `minikube kubectl -- get pods` if you want to avoid explicit context switching.
* Backup your kubeconfig before editing.
* Avoid sharing your kubeconfig with credentials in public repos.

---

## 8. Commands Cheat Sheet (Minikube-Focused)

```bash
# Start Minikube
minikube start

# Show kubeconfig
kubectl config view

# Show current context
kubectl config current-context

# List all contexts
kubectl config get-contexts

# Switch context
kubectl config use-context minikube

# Set default namespace
kubectl config set-context --current --namespace=dev

# Test kubectl
kubectl get pods -A
```

---

### Beginner Takeaways

* kubeconfig **organizes cluster access** in one file.
* Minikube automatically generates kubeconfig for you.
* Contexts allow **easy switching between clusters or namespaces**.
* Safe access control and reproducible environments depend on correct kubeconfig usage.
